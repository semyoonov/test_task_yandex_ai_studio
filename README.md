## 📁 Структура проекта

*   [Описание работы.pdf](Описание%20работы.pdf) — отчет с дизайном экспериментов и описанием решения.

#### Остальные файлы:
*   `dataset/` — папка с исходными данными.
*   `generate_data.py` — скрипт для предобработки данных и подготовки датасета.
*   `train.py` — скрипт обучения модели и оценки результатов:
    *   Проводит подбор гиперпараметров модели, обучает её.
    *   Определяет оптимальный порог классификации для максимизации $F_2-score$.
    *   Сравнивает качество работы старой системы и ML-модели на тестовом периоде.

## 📊 Метрики

Для оценки моделей используется разделение по времени: обучение производится на данных за 2022–2023 гг., а тестирование — на данных за 2024 г. 

В качестве основной бизнес-метрики выбран **$F_2-score$**, так как полнота охвата аварий (Recall) для энергосистемы в 2 раза важнее точности (Precision).

| Модель | ROC-AUC | Recall | $F_2$-score |
| :--- | :---: | :---: | :---: |
| Бейзлайн (Пороговая модель) | 0.5317 | 10.0% | 0.0921 |
| **ML-модель (HistGB + GridSearch)** | **0.6845** | **39.0%** | **0.1862** |

**Выводы**:
1. Переход на машинное обучение увеличил охват улавливаемых аварий (Recall) **почти в 4 раза** (с 10% до 39%).
2. Целевой показатель $F_2$-score вырос **более чем в 2 раза** (с 0.092 до 0.186).
3. ROC-AUC модели поднялся с уровня случайного гадания (0.53) до стабильного значения 0.68.
